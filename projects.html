Online Judge

The project is aimed at building the backend of a web based Judge which checks a code for Correctness, Run time and memory. Haskell is the ideal language for this since we can effectively use Concurrent threads and memory to lazily compute everything. Further this can be deployed using web based frameworks such as Yesod. Here again Haskell is beneficial since it is comparitively faster to program scripts which check for malicious code in Haskell. View the code at Github.com

Autonomous Underwater Vehicle

The project is aimed at creating an Autonomous Robot which runs underwater. The programming problem can be divided into 3 main sections which are Controls, Image Processing and Aritificial Intelligence algorithms. Presently we use Robot OS' effective message passing, goal callback and effective unit testing techniques to optimize the controls and seperating the logical levels from the actuation levels. Further we use OpenCV for image processing.
View the code at Github.com

Connec4AI

The basic idea is to effectively use the laziness of Haskell to easily find the winning strategy in a given game of TicTacToe without overflowing the stacks. The next step would be to implement the same for Connect4 but using probabilistic analysis on the side, as a complement to further optimize the process for larger dimensions. The frontend implementation for the game is still under consideration.
